1 Why we are doing this

Train ourselves before we train the AI. Phase 1 is the team running the workflow manually so we validate the structure on real requests. Once it works with humans, the AI Agent layers on top in Phase 2.

⚡
Fast path for simple, clear requests If the PDM is confident the request is simple and clear, they can skip the separate requirements-approval round trip. PDM posts a quick confirmation in Basecamp ("just to make sure I'm interpreting correctly, you want X — DM will follow up shortly with estimate and timing"), then signals the DM to proceed straight to estimation. The DM's reply (requirements + estimate + timing + release train) becomes the single client approval moment instead of two.

4 What's different starting now

DM
Publish service budget bucket hours per account Planning/Ops/New Requests — covers standing team meetings (dailies, planning, retros, client status calls); remaining hours fund new request processing. Production Support — processing reported issues pull from here. Implementation — approved work pulls from here once requirements, estimate, and timing are signed off. DM tracks continuously as time is logged.
PDM
Create the Zoho request ticket PDM sets at creation: Item Type (Task / Bug) · Epic · Priority (aligned with client) · Request Status · Assigned to DM. Status follows ownership — whoever the ticket is assigned to during the current phase updates the Status as the request moves through analysis, scope, estimation, and approval. Tag handoffs in comments and Slack.
All
Run the corporate site as our pilot PDMs submit, DMs process. Surface what doesn't work. Refine before any of this goes live on a client account.

5 The three tools we operate in

BASECAMP
Where the client lives. Requests captured, scope authored, scope approved.
ZOHO
Ticket lives in the backlog during the front half — PDM updates the Request Analysis Status field as work progresses. Once Scope Approved, the ticket moves to Sprint Candidates and the active board takes over.
SMARTSHEET
Client visibility. Request tracker, project plan when needed, request estimator.
